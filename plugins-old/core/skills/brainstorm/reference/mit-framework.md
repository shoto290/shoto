# MIT Brainstorming Framework

Canonical source for the rules and process referenced by `brainstorm`. Adapted from the MIT Human Resources brainstorming guide.

## The 4 rules

Surface all four to the user during Phase 1 (Frame). They govern Phase 2.a (Diverge) without exception.

| # | English | Français |
|---|---------|----------|
| 1 | All ideas are welcome. | Toutes les idées sont les bienvenues. |
| 2 | No judgment during the brainstorm. | Pas de jugement pendant le brainstorm. |
| 3 | More ideas the better — go for quantity. | Plus il y a d'idées, mieux c'est — viser la quantité. |
| 4 | Duplicates are OK. | Les doublons sont acceptés. |

Rule 2 is the load-bearing one: every evaluation, critique, or hedge during divergence violates the contract. Park evaluation for Phase 2.c.

## The 7-step process

1. **Introduce the session and the rules** — explain what brainstorming is and announce the four rules.
2. **Clarify the desired outcomes** — what does success look like at the end of the session?
3. **Display the question** — write the question where everyone can see it; keep it visible the whole time.
4. **One minute of silent reflection** — let participants think before speaking.
5. **Generate ideas** — open round of free idea generation, respecting the four rules.
6. **Number and organize** — number every idea, then cluster by theme.
7. **Clarify** — ask the author of any unclear idea to explain it so the group can evaluate later.

## Mapping to `brainstorm` phases

| MIT step | `brainstorm` phase |
|----------|-------------------|
| 1. Introduce + rules | Phase 1 — Frame (surface the 4 rules) |
| 2. Clarify desired outcomes | Phase 1 — Frame (restate the question) |
| 3. Display the question | Phase 1 — Frame (one-sentence restatement) |
| 4. Silent reflection | Implicit in Claude's generation step before Phase 2.a |
| 5. Generate ideas | Phase 2.a — Diverge |
| 6. Number and organize | Phase 2.b — Organize |
| 7. Clarify | Phase 2.c — Converge (light) + Phase 2.d decision |

The 7 MIT steps collapse cleanly into the Frame → Diverge → Organize → Converge → Decide loop used by the skill.
