The `git` plugin automates the full pull-request lifecycle inside Claude Code. Use it to commit changes, open a conventionally-titled PR, triage review comments into a structured decision list, and apply only the approved fixes — all without leaving your editor.

## Overview

Four skills form a linear pipeline:

1. **`git:commit`** — stages current changes (skipping secrets), infers a Conventional Commit title, confirms with you, then creates exactly one commit without pushing.
2. **`git:create`** — reads your commits and diff, drafts a Conventional Commit title and a non-technical changelog body, confirms with you, then pushes and opens the PR.
3. **`git:review-comments`** — reads PR comments (pasted inline) and returns a verdict list (`FIX` / `FIX-STYLE` / `INTENTIONAL` / `OUT-OF-SCOPE` / `DISCUSS`). No files are touched.
4. **`git:review-fix`** — consumes that verdict list and applies only `FIX` and `FIX-STYLE` items, one at a time, then runs auto-detected verification (tests / lint / typecheck).

The `reviewer` sub-agent is auto-delegated by `git:review-comments` to evaluate individual comments in parallel when there are four or more.

## Install

```bash
/plugin marketplace add shoto290/shoto
/plugin install git@shoto
```

After install, run `/help` to confirm the `git:` skills are listed and `/agents` to confirm the `reviewer` sub-agent is registered.

<Note>
  `git:create` requires the `gh` CLI (GitHub CLI) to be installed and authenticated. Run `gh auth status` to verify. If authentication fails, run `gh auth login` before invoking the skill.
</Note>

## Typical flow

<Steps>
  <Step title="Commit your changes">
    Run `/git:commit`. The skill stages your current changes, confirms a conventional title with you, and creates a single commit. Re-invoke for each additional logical commit.
  </Step>
  <Step title="Open the PR">
    Run `/git:create`. The skill reads your commits, drafts a title and body, asks for confirmation, then pushes and creates the PR.
  </Step>
  <Step title="Triage review comments">
    Paste the PR comments and run `/git:review-comments`. You get a verdict block per comment and a summary table. No files are changed.
  </Step>
  <Step title="Apply fixes">
    Paste the verdict list output into `/git:review-fix`. Only `FIX` and `FIX-STYLE` items are applied. Verification runs automatically at the end.
  </Step>
</Steps>
