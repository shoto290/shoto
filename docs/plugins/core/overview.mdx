The `core` plugin is the meta-tooling layer of the shoto marketplace. It gives you expert skills and two architect sub-agents that know the Claude Code authoring conventions cold — so you can build new capabilities without hand-crafting every artifact from scratch.

**Version:** 0.11.0 — **Depends on:** [`utils`](/plugins/utils/overview)

## Install

```bash
/plugin install core@shoto
```

After install, run `/help` to see the `core:` skills and `/agents` to confirm the two architect sub-agents are registered.

## Typical flow

<Steps>
  <Step title="Describe the capability">
    Run `/core:evolve add a CI-failure triage workflow`. The skill infers what you need from the conversation if you omit the argument.
  </Step>
  <Step title="Review the plan">
    `evolve` inventories existing artifacts via `explore:explore`, surfaces overlaps, and proposes a dependency-ordered plan. You confirm or adjust before anything is written.
  </Step>
  <Step title="Delegation and execution">
    Approved skill entries go to `skill-architect`, sub-agent entries to `subagent-architect`, hook entries to the `hooks` skill. Architects run in parallel when their work is independent.
  </Step>
  <Step title="Restart and test">
    New skill directories and new sub-agent files require a Claude Code restart. In-place edits to existing `SKILL.md` files take effect immediately.
  </Step>
</Steps>
