---
title: "The `skills-list` skill"
description: "List every skill installed in the current context — global, project-local, and plugin cache — grouped by source."
---

`/core:skills-list` answers "what skills are available to me right now?" by enumerating every source Claude Code checks — the global user directory, the current project, and the marketplace plugin cache — and producing a compact grouped report. It is strictly read-only.

<Note>
  This skill lists and describes installed skills. To install a new skill, use `find-skills`. To get a recommendation for a specific task, use [`core:skills-suggest`](/plugins/core/skills/skills-suggest).
</Note>

## Overview

The skill reads only the first 10 lines of each `SKILL.md` it finds — enough to extract `name` and `description` from the frontmatter — so it stays fast even with a large number of installed skills. If the current repo is itself a Claude Code marketplace (i.e. `.claude-plugin/marketplace.json` exists at the root), a fourth source — local marketplace plugins — is also enumerated.

## When to invoke it

Trigger phrases:

- `/core:skills-list`
- `list my skills`
- `what skills are installed`
- `show installed skills`
- `what's available in this repo?`
- `where am I in terms of skills?`

## How it works

<Steps>
  <Step title="Detect the current context">
    Runs `pwd` to capture the working directory, then `git rev-parse --show-toplevel` to find the enclosing repo root. If the repo root contains `.claude-plugin/marketplace.json`, the local marketplace source is also scanned.
  </Step>
  <Step title="Enumerate sources">
    Uses `Glob` to find `SKILL.md` files under four paths and reads only the first 10 lines of each to extract `name` and `description`:

    | Source | Path |
    | :-- | :-- |
    | Global (user) | `~/.claude/skills/*/SKILL.md` |
    | Project (current repo) | `.claude/skills/*/SKILL.md` |
    | Plugin cache (marketplace) | `~/.claude/plugins/cache/**/skills/*/SKILL.md` |
    | Local marketplace (if applicable) | `plugins/*/skills/*/SKILL.md` |
  </Step>
  <Step title="Group and de-duplicate">
    Groups results by source. If the same skill `name` appears in more than one source, it is listed under each source and flagged as a conflict. This matters when developing a plugin that is also installed via the cache.
  </Step>
  <Step title="Render the report">
    Produces a markdown table per source with columns `Skill`, `Plugin`, and `Description (one line, ≤80 chars)`. Ends with a summary line of counts.
  </Step>
</Steps>

## Output format

```markdown
### Global (`~/.claude/skills/`)

| Skill          | Plugin | Description                                                       |
| -------------- | ------ | ----------------------------------------------------------------- |
| note-taker     | —      | Capture quick notes into the daily journal                        |

### Project (`.claude/skills/`)

| Skill         | Plugin | Description                                                       |
| ------------- | ------ | ----------------------------------------------------------------- |
| release-notes | —      | Draft release notes from conventional commits                     |

### Plugin cache (`~/.claude/plugins/cache/`)

| Skill        | Plugin           | Description                                                       |
| ------------ | ---------------- | ----------------------------------------------------------------- |
| base         | shoto/core@0.8.0 | Canonical rules every skill in the marketplace inherits from      |
| skills-list  | shoto/core@0.8.0 | List the skills installed and available in the current context    |

2 global · 1 project · 2 plugin-cache · 5 total skills available in this context
```

## Anti-patterns

- Don't use this skill to install a new skill — that's `find-skills`.
- Don't use this skill to recommend a skill for a task — that's [`core:skills-suggest`](/plugins/core/skills/skills-suggest).
- Don't read full `SKILL.md` bodies — only the frontmatter (first 10 lines) is needed.
- Don't modify any skill files — this skill is read-only.

## Reference

- [`core:base` section 5](/plugins/core/skills/base) — canonical delegation map; `skills-list` is one of the entries in it.
- [`core:skills-suggest`](/plugins/core/skills/skills-suggest) — uses the output of this skill internally to filter recommendations to actually installed skills.
- `find-skills` — for installing new skills into the current context.
