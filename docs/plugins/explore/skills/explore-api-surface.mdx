<Note>
  Internal specialist — dispatched only by [`/explore:explore`](/plugins/explore/skills/explore). Not invokable as a standalone slash command. Documented here so you can understand how the orchestrator routes requests.
</Note>

`/explore-api-surface` produces a focused public-surface map for a module — all exports, HTTP/RPC endpoints, and public types, with sampled callers per symbol and a list of dead public symbols that have no callers. It runs in fork mode via the `api-surface-explorer` sub-agent and returns only the anchored report.

## When to use it

Trigger phrases that route to this skill:

- `api`, `public api`, `public surface`, `surface`
- `exports`, `endpoints`, `interface`, `contrat`
- `consumers`, `breaking change`

It is dispatched automatically by `/explore` when your topic contains any of those keywords. You can also force it with `profile=api-surface`:

```text
/explore profile=api-surface <module path or feature>
```

It is also used as part of the `"before breaking <module>"` multi-specialist heuristic alongside `flow`.

## What the skill looks for

- **Public exports** — barrel files, top-level module exports, `__all__` (Python), `pub` items (Rust), default and named exports (TS/JS).
- **HTTP/RPC endpoints** — route registrations (Express, Fastify, Koa, Flask, FastAPI, Echo), OpenAPI specs, proto files, GraphQL schemas. Each mapped to its handler.
- **Signatures of public functions and types** — one line each; argument types and return type included when non-obvious from the name.
- **Sampled consumers** — 1–3 callers per public symbol, capped at 20 total consumer entries.
- **Dead public symbols** — exported but zero callers anywhere in the repo; candidates for removal (prefixed with `?` when external consumer uncertainty exists).

## Report format

```markdown
## Exports
- <path>:<line> — `<signature>` — <<=10-word role>

## HTTP/RPC
- <method> <path> — handled at <path>:<line>

## Public types
- <type name> — declared at <path>:<line>

## Consumers (sampled)
- <symbol> ← <caller path>:<line>

## Dead or risky
- <symbol> at <path>:<line> — <reason>
```

All five sections are always emitted. An empty section is written as `- (none)`.

## Budget and rules

| Constraint | Limit |
| :-- | :-- |
| Files opened | 15 max |
| Report length | 55 lines max |
| Consumer entries | 20 max total |
| Anchors | Required on every fact |
| Unknown facts | Prefixed with `?` |

The skill does not quote source code and does not narrate. Every claim must be anchored.

## Sub-agent

This skill delegates to the [`api-surface-explorer`](/plugins/explore/agents/api-surface-explorer) sub-agent, which runs in a forked context. The report returned to you is the sub-agent's output verbatim.
