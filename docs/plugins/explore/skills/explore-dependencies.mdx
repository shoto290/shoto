</Note>

`/explore-dependencies` produces a focused dependency map for a directory or package — which external libraries are in use, how packages couple internally, where direction violations occur, and which declared dependencies are unused or oversized. It runs in fork mode via the `dependencies-explorer` sub-agent and returns only the anchored report.

## When to use it

Trigger phrases that route to this skill:

- `dependency`, `depends on`, `coupling`, `dépend de`
- `imports`, `uses lib`, `external deps`, `package coupling`
- `couplage` (French equivalent)

It is dispatched automatically by `/explore` when your topic contains any of those keywords. You can also force it with `profile=dependencies`:

```text
/explore profile=dependencies <directory or package>
```

It is also used as part of two multi-specialist heuristics: `"audit <area> for ship readiness"` (alongside `tests`) and `"refactor <area>"` (alongside `architecture`).

## What the skill looks for

- **External libraries** — imports found in the area, cross-referenced against the package manifest (`package.json`, `requirements.txt`, `go.mod`, `Cargo.toml`, etc.). Purpose noted in `<=8` words per lib.
- **Internal coupling** — for monorepos: which other packages this area imports. For single-package: cross-module imports (e.g. `src/auth → src/db`).
- **Direction violations** — imports that go against the area's expected layering. If a layering convention is detectable (folder names like `domain/`, `infrastructure/`) violations are flagged directly; otherwise a directory-depth heuristic applies and results are prefixed with `?`.
- **Unused or oversized** — declared in the manifest but not imported anywhere (unused), or a large library used for one trivial function (oversized).

## Report format

```markdown
## External deps
- <lib> — used at <path>:<line> — <<=8-word purpose>

## Internal coupling
- <pkg-A> → <pkg-B> at <path>:<line>

## Direction violations
- <from> → <to> at <path>:<line> — <<=10-word reason>

## Unused or oversized
- <lib> — <reason> (<path>:<line> | manifest)
```

All four sections are always emitted. An empty section is written as `- (none)`.

## Budget and rules

| Constraint | Limit |
| :-- | :-- |
| Files opened | 20 max |
| Report length | 60 lines max |
| Anchors | Required on every fact (`manifest` accepted for declared-but-unused) |
| Unknown facts | Prefixed with `?` |

The skill does not quote source code and does not narrate. Every claim must be anchored.

## Sub-agent

This skill delegates to the [`dependencies-explorer`](/plugins/explore/agents/dependencies-explorer) sub-agent, which runs in a forked context. The report returned to you is the sub-agent's output verbatim.
