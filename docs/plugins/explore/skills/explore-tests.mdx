<Note>

</Note>

`/explore-tests` produces a focused test-coverage map for a codebase area — which symbols are covered, which are likely uncovered, what test patterns are in use, and where the notable gaps are. It runs in fork mode via the `tests-explorer` sub-agent and returns only the anchored report.

## When to use it

Trigger phrases that route to this skill:

- `test`, `tests for`, `is this tested`, `coverage`
- `tested`, `untested`, `smoke test`
- `couvert`, `testé` (French equivalents)
- `what is tested in <area>`

It is dispatched automatically by `/explore` when your topic contains any of those keywords. You can also force it with `profile=tests`:

```text
/explore profile=tests <area>
```

It is also used as part of the `"audit <area> for ship readiness"` multi-specialist heuristic alongside `dependencies`.

## What the skill looks for

- **Test file locations** — files colocated with source or under `tests/` / `__tests__/`, language-appropriate conventions (`*.test.ts`, `*.spec.py`, `*_test.go`, `test_*.py`).
- **Source ↔ test mapping** — matches by filename pair (e.g. `Foo.ts` ↔ `Foo.test.ts`) or by import (test file imports the source module).
- **Uncovered public symbols** — exported functions, classes, types, and endpoints with no matching test file and no import from any test file.
- **Test patterns** — framework in use (Vitest, Jest, Pytest, Go test, etc.), unit/integration/e2e split, mock strategy (msw, nock, fakes, fixtures). A pattern is only reported when it appears `>=3` times.
- **Notable gaps** — error paths never tested, public APIs without a smoke test, recently changed files without test updates (when `git log` is accessible).

## Report format

```markdown
## Covered
- <symbol> at <path>:<line> ← tested at <test path>:<line>

## Uncovered (likely)
- <symbol> at <path>:<line> — <why>

## Test patterns
- <pattern> — sample at <test path>:<line>

## Notable gaps
- <gap> (<path>:<line>)
```

All four sections are always emitted. An empty section is written as `- (none)`.

## Budget and rules

| Constraint | Limit |
| :-- | :-- |
| Files opened | 18 max |
| Report length | 55 lines max |
| Anchors | Required on every fact |
| Unknown facts | Prefixed with `?` |

The skill does not quote source code and does not narrate. Every claim must be anchored.

## Sub-agent

This skill delegates to the [`tests-explorer`](/plugins/explore/agents/tests-explorer) sub-agent, which runs in a forked context. The report returned to you is the sub-agent's output verbatim.
