---
title: "The `explore-component` skill"
description: "Find existing components, utilities, or services that match a desired capability, score reuse fit, and get a REUSE / EXTEND / NEW recommendation."
---

`/explore-component` answers the question "does something for this already exist?" It runs in fork mode via the `component-explorer` sub-agent, scores every candidate by fit, identifies gaps, and returns one clear recommendation — `REUSE`, `EXTEND`, or `NEW` — backed by anchored evidence.

## When to use it

Trigger phrases that load this skill:

- `/explore-component <capability you want to add>`
- "does X already exist in this codebase"
- "is there a component I can reuse for Y"
- "can I extend Z instead of building new"
- "avoid duplicating logic"

It is also dispatched automatically by `/explore` when your topic contains keywords such as `reuse`, `existing component`, `duplicate`, `can I use`, `is there a`, or the French equivalents.

## Usage

```text
/explore-component <need or capability you want to add>
```

## Fit scoring

| Score | Meaning |
| :-- | :-- |
| `high` | ≥80% of requested sub-capabilities already covered |
| `medium` | 40–79% covered |
| `low` | `<40%` covered |

## Recommendation rules

| Recommendation | When it applies |
| :-- | :-- |
| `REUSE <component>` | At least one high-fit candidate with no missing sub-capabilities |
| `EXTEND <component> (add <gap>)` | Best candidate is high or medium fit; 1–2 gaps fit its surface |
| `NEW (no good match)` | No candidate above low fit, or the best candidate's surface is wrong for the gaps |

## Report format

```markdown
## Matches
- <path>:<line> — `<signature>` — <10-word role> — fit: high|medium|low

## Reuse fit
- <component> covers: <comma-separated capabilities>
- <component> missing for this need: <comma-separated gaps>

## Gaps
- <capability> — closest workaround at <path>:<line>

## Recommendation
<REUSE <component> | EXTEND <component> (add <gap>) | NEW (no good match)>
```

All four sections are always emitted. An empty section is written as `- (none)`.

## Budget and rules

| Constraint | Limit |
| :-- | :-- |
| Files opened | 15 max |
| Report length | 50 lines max |
| Anchors | Required on every fact |
| Unknown facts | Prefixed with `?` |

## Sub-agent

This skill delegates to the [`component-explorer`](/plugins/explore/agents/component-explorer) sub-agent, which runs in a forked context. The report returned to you is the sub-agent's output verbatim.
