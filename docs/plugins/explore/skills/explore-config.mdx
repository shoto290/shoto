<Note>
  Internal specialist — dispatched only by [`/explore:explore`](/plugins/explore/skills/explore). Not invokable as a standalone slash command. Documented here so you can understand how the orchestrator routes requests.
</Note>

`/explore-config` surfaces all the runtime knobs a feature reads from its environment — env vars, config files, and feature flags — and flags any value that has no documented default or schema. It runs in fork mode via the `config-explorer` sub-agent and returns only the anchored report.

## When to use it

Trigger phrases that route to this skill:

- `config`, `configuration`, `env var`, `environment variable`
- `feature flag`, `settings`, `runtime knob`, `toggle`, `.env`

It is dispatched automatically by `/explore` when your topic contains any of those keywords. You can also force it with `profile=config`:

```text
/explore profile=config <feature or area>
```

It is also used as part of the `"onboard onto <area>"` multi-specialist heuristic alongside `architecture`.

## What the skill looks for

- **Env var references** — `process.env.X` (JS/TS), `os.environ[...]` / `os.getenv` (Python), `os.Getenv` (Go), `std::env::var` (Rust), `${VAR}` in shell scripts. Scoped to the requested area.
- **Config file consumers** — code that reads `.json`, `.yaml`, `.yml`, `.toml`, `.ini`, or `.env`-style files. Each consumer is mapped to the config file path.
- **Feature flag SDK calls** — LaunchDarkly, GrowthBook, Split, Statsig, Unleash, ConfigCat, OpenFeature. Flag name and call site captured.
- **Cross-reference with declarations** — vars are checked against `.env.example`, schema files, and Zod/Joi/Pydantic settings classes. Vars with no declaration are marked `?`.
- **Undocumented or risky values** — consumed without a default fallback and without a documented declaration; secret-like names (`*_KEY`, `*_TOKEN`, `*_SECRET`) accessed at runtime without comment. Actual secret values are never printed — replaced with `<redacted>`.

## Report format

```markdown
## Env vars
- <NAME> — consumed at <path>:<line> — <<=8-word purpose>

## Config files
- <config path> → loaded at <code path>:<line>

## Feature flags
- <flag name> — checked at <path>:<line> — SDK: <name>

## Undocumented / risky
- <name> — <reason> (<path>:<line>)
```

All four sections are always emitted. An empty section is written as `- (none)`.

## Budget and rules

| Constraint | Limit |
| :-- | :-- |
| Files opened | 15 max |
| Report length | 55 lines max |
| Anchors | Required on every fact |
| Unknown facts | Prefixed with `?` |
| Secret values | Never printed — replaced with `<redacted>` |

The skill does not quote source code and does not narrate. Every claim must be anchored.

## Sub-agent

This skill delegates to the [`config-explorer`](/plugins/explore/agents/config-explorer) sub-agent, which runs in a forked context. The report returned to you is the sub-agent's output verbatim.
