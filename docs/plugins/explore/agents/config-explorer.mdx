The `config-explorer` sub-agent is a focused, read-only configuration specialist. It surfaces all the runtime knobs a feature reads from its environment — env vars, config files, and feature flags — so onboarding, debugging, and audits are easier. It never modifies code. It is spawned by `explore-config` (and by the `explore` orchestrator when routing to the config specialist).

## What it does

<Steps>
  <Step title="Find env var references">
    Scans the area for `process.env.X` (JS/TS), `os.environ[...]` / `os.getenv` (Python), `os.Getenv` (Go), `std::env::var` (Rust), and `${VAR}` patterns in shell scripts.
  </Step>
  <Step title="Find config file consumers">
    Locates code that reads `.json`, `.yaml`, `.yml`, `.toml`, `.ini`, or `.env`-style files. Maps each consumer to the config file path it loads.
  </Step>
  <Step title="Find feature flag SDK calls">
    Matches common SDK usage by import or function name — LaunchDarkly (`ldClient.variation`), GrowthBook (`gb.getFeatureValue`), Split (`splitClient.getTreatment`), Statsig (`Statsig.getValue`), Unleash, ConfigCat, OpenFeature. Captures the flag name and call site.
  </Step>
  <Step title="Cross-reference with declarations">
    Checks env vars against `.env.example`, schema files, and Zod/Joi/Pydantic settings classes. Vars with no declaration are marked `?`.
  </Step>
  <Step title="Flag undocumented or risky values">
    Values consumed without a default fallback and without a documented declaration are flagged. Secret-like names (`*_KEY`, `*_TOKEN`, `*_SECRET`) accessed at runtime without a comment are also flagged. Actual secret values are never printed — replaced with `<redacted>`.
  </Step>
</Steps>

## Report format

```markdown
## Env vars
- <NAME> — consumed at <path>:<line> — <<=8-word purpose>

## Config files
- <config file path> → loaded at <code path>:<line>

## Feature flags
- <flag name> — checked at <path>:<line> — SDK: <name>

## Undocumented / risky
- <name> — <reason> (<path>:<line>)
```

All four sections are always present. Empty sections use `- (none)`. Unknown facts are prefixed with `?`.

## Budget and rules

| Constraint | Value |
| :-- | :-- |
| Files opened | 15 max |
| Report length | 55 lines max |
| Tools available | `Read`, `Glob`, `Grep`, `Bash` |
| Model | sonnet |
| Secret values | Never printed — replaced with `<redacted>` |

The sub-agent never quotes source code and never narrates. Every claim must carry a `path:line` anchor.

## Invocation

This sub-agent is paired with and invoked by the [`explore-config`](/plugins/explore/skills/explore-config) skill. The skill runs in fork mode: the sub-agent explores in its own context window and returns only the report.
