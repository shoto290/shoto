The `convention-explorer` sub-agent is a focused, read-only conventions specialist. It maps the implicit style rules of a codebase area — naming, folder structure, idioms, and error/logging/test patterns — so a future writer can produce code that fits. It never modifies code. It is spawned by `explore-convention` (and by the `explore` orchestrator when routing to the convention specialist).

## What it does

<Steps>
  <Step title="Sample representative files">
    Globs the area and picks 8–12 files from different sub-folders to avoid bias toward a single module.
  </Step>
  <Step title="Extract naming patterns">
    Only reports a pattern as a convention when it appears 3 or more times across distinct files. Covers filenames, classes/types, functions/methods, variables/constants, and test files.
  </Step>
  <Step title="Detect folder/file organization rules">
    Identifies patterns such as colocated tests, barrel `index.ts` files, one component per folder, and module boundaries.
  </Step>
  <Step title="Identify idioms">
    Surfaces language or framework preferences the codebase consistently applies — e.g., `Result<T,E>` instead of throws, factory functions over classes, Zod schemas at the module boundary.
  </Step>
  <Step title="Capture style samples">
    Records one anchor each for error handling, logging, and tests. Notes the type or class used, the call-site shape, and any wrapping pattern.
  </Step>
  <Step title="Flag inconsistencies">
    When two competing patterns coexist, lists both with anchors and prefixes each with `?`.
  </Step>
</Steps>

## Report format

```markdown
## Naming
- <kind: files | classes | functions | vars | tests> → <pattern> — sample at <path>:<line>

## Structure
- <folder or rule> → <role> — example at <path>:<line>

## Idioms
- <idiom> — sample at <path>:<line>

## Style
- error handling: <pattern> (<path>:<line>)
- logging: <pattern> (<path>:<line>)
- tests: <pattern> (<path>:<line>)
```

All four sections are always present. Empty sections use `- (none)`. Conflicting conventions appear as two entries with a `?` prefix.

## Budget and rules

| Constraint | Value |
| :-- | :-- |
| Files opened | 15 max |
| Files sampled | 8–12, from different sub-folders |
| Report length | 55 lines max |
| Tools available | `Read`, `Glob`, `Grep`, `Bash` |
| Model | sonnet |

The sub-agent never quotes source code and never narrates. Every claim must carry a `path:line` anchor.

## Invocation

This sub-agent is paired with and invoked by the [`explore-convention`](/plugins/explore/skills/explore-convention) skill. The skill runs in fork mode: the sub-agent explores in its own context window and returns only the report.
