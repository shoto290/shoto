The `architecture-explorer` sub-agent is a focused, read-only structural analyst. It opens files only to confirm patterns and never modifies code. It is spawned by `explore-architecture` (and by the `explore` orchestrator when routing to the architecture specialist).

## What it looks for

<Steps>
  <Step title="Map directory shape">
    Uses `Glob` to survey the directory structure and `Grep` for recurring symbols — class names matching `.*Repository`, `.*Service`, framework annotations, DI tokens — before opening any file.
  </Step>
  <Step title="Identify recurring patterns">
    Looks for layered, hexagonal, clean, MVC, MVVM, repository, factory, observer, strategy, mediator, event-bus, CQRS, and port-adapter shapes. Only reports a pattern when it appears at least 3 times across distinct files.
  </Step>
  <Step title="Map layers">
    For each architectural layer found, names it, states its responsibility in 10 words or fewer, and anchors an entry-point file.
  </Step>
  <Step title="Surface conventions">
    Structural rules that hold across the area — e.g. "one aggregate root per folder", "controllers thin, services thick". Each convention requires an example anchor.
  </Step>
  <Step title="Flag anti-patterns">
    Reports god-object, circular dependencies between layers, business logic in controllers, and hidden global state — only when 2 or more concrete occurrences exist.
  </Step>
</Steps>

## Report format

```markdown
## Patterns
- <pattern name> — applied at <path>:<line> — <10-word rationale>

## Layers
- <layer> — <10-word responsibility> — entry at <path>:<line>

## Conventions
- <convention> — exemplified at <path>:<line>

## Anti-patterns / smells
- <issue> (<path>:<line>)
```

All four sections are always present. Empty sections use `- (none)`. Unknown facts are prefixed with `?`.

## Budget and rules

| Constraint | Value |
| :-- | :-- |
| Files opened | 20 max |
| Report length | 60 lines max |
| Tools available | `Read`, `Glob`, `Grep`, `Bash` |
| Model | sonnet |

The sub-agent never quotes source code and never narrates. Every claim must carry a `path:line` anchor.

## Invocation

This sub-agent is paired with and invoked by the [`explore-architecture`](/plugins/explore/skills/explore-architecture) skill. The skill runs in fork mode: the sub-agent explores in its own context window and returns only the report.
