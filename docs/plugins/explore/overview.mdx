The `explore` plugin gives you focused, read-only intelligence about an unfamiliar codebase area. One orchestrator skill parses your intent and dispatches the right specialist — or several in parallel — each of which runs in fork mode and returns a compact, location-anchored report.

## Overview

Four specialist pairs each cover a distinct lens:

| Specialist | Question it answers |
| :-- | :-- |
| **architecture** | What structural patterns, layers, and anti-patterns exist? |
| **component** | Does a component for this capability already exist? |
| **flow** | What is the call chain and data flow for this feature? |
| **convention** | What naming, idiom, and style rules does this area follow? |

Each pair is a skill + sub-agent: the skill defines the output contract and budget; the sub-agent does the file reading.

When you invoke `/explore <topic>`, the orchestrator detects intent from keywords and dispatches automatically. For broad questions it can run multiple specialists in parallel.

## Entry point

`/explore <topic>` is the only command you invoke directly. The orchestrator parses your phrasing and routes to the right specialist — or fans out to several at once for broad questions.

<Note>
  If no specialist keyword is detected, the orchestrator falls back to general exploration mode (`survey`, `deep-dive`, or `targeted`) and handles the request inline without delegating to a sub-agent.
</Note>

### Internal specialists dispatched by the orchestrator

The four specialist skills below are not invoked directly. The orchestrator selects and dispatches them based on intent detection. Their pages are documented here so you can understand how routing works.

- [`explore-architecture`](/plugins/explore/skills/explore-architecture) — structural patterns, layer boundaries, anti-patterns
- [`explore-component`](/plugins/explore/skills/explore-component) — reuse fit scoring, REUSE / EXTEND / NEW recommendation
- [`explore-flow`](/plugins/explore/skills/explore-flow) — call chain and data flow from entry point to exits
- [`explore-convention`](/plugins/explore/skills/explore-convention) — naming patterns, folder structure, idioms, style samples

## Sub-agents

<CardGroup cols={2}>
  <Card title="architecture-explorer" href="/plugins/explore/agents/architecture-explorer">
    Reads up to 20 files to map patterns, layers, conventions, and anti-patterns.
  </Card>
  <Card title="component-explorer" href="/plugins/explore/agents/component-explorer">
    Reads up to 15 files, scores fit per candidate, and returns a single REUSE / EXTEND / NEW recommendation.
  </Card>
  <Card title="flow-explorer" href="/plugins/explore/agents/flow-explorer">
    Reads up to 20 files, traces up to 5 hops deep, and returns entry, steps, side effects, and exit points.
  </Card>
  <Card title="convention-explorer" href="/plugins/explore/agents/convention-explorer">
    Samples 8–12 files across sub-folders and returns naming, structure, idioms, and style sections.
  </Card>
</CardGroup>
