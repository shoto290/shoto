The `docs` plugin gives you three skills and one sub-agent for building and maintaining Mintlify documentation sites from inside Claude Code. It covers the full cycle: scaffold a new site, author MDX pages from prose or from source code, and validate the build before committing.

<Note>
  The Mintlify site you are reading right now was scaffolded and authored using this plugin.
</Note>

## Overview

The plugin is organized around a single linear workflow:

1. **`mintlify-init`** — run `mint new` and patch `docs.json` with your site name, color, and GitHub link.
2. **`mintlify-page`** — add a new MDX page to an existing group, optionally generating content from source code.
3. **`mintlify-validate`** — run `mint validate` and `mint build`, then report every broken link, invalid frontmatter entry, and orphan file.

The `docs-architect` sub-agent operates behind the scenes for page creation and audits. You can also invoke it directly for multi-page refactors.

**Version:** `0.2.0`  
**Prerequisites:** `mint` CLI must be on your PATH. Install it with:

```bash
npm i -g mint
```

## Install

```bash
/plugin marketplace add shoto290/shoto
/plugin install docs@shoto
```

After install, run `/help` to see the `docs:` skills and `/agents` to confirm `docs-architect` is registered.
