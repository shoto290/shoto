The `bench` plugin (v0.1.0) lets you benchmark skill and sub-agent behavior over time. It discovers test files colocated with your artifacts, runs them in parallel, and writes diff-friendly markdown reports.

## Overview

`bench` solves a specific problem: you change a skill or sub-agent prompt and have no way to know whether it improved or regressed. By colocating `tests/*.md` files with each artifact, you get a repeatable, version-controlled record of how each artifact behaves.

The `/bench:run` skill orchestrates everything — it discovers tests, asks which artifact and tests to run, spawns one `bench-runner` sub-agent per test in parallel, captures wall-clock timing, and writes a report under `runs/<artifact>/<timestamp>.md`.

**Version:** `0.1.0`

## Install

```bash
/plugin install bench@shoto
```

If you haven't added the shoto marketplace yet, run this first:

```bash
/plugin marketplace add shoto290/shoto
```

## Test file location

Tests live alongside the artifact they exercise:

```
plugins/<plugin>/skills/<name>/tests/*.md
plugins/<plugin>/agents/<name>/tests/*.md
```

Each `*.md` file in a `tests/` directory is treated as one benchmark case.
