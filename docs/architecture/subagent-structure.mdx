title: "Sub-Agent Structure"
description: "A single .md file per agent: YAML frontmatter declares identity and constraints; the markdown body is the full system prompt."
---

A sub-agent is a Markdown file with YAML frontmatter. It runs in its own context window with a custom system prompt, configurable tool set, and independent permissions. Only the summary it produces lands in the main conversation — intermediate tool calls stay isolated.

## File Layout

```markdown
---
name: code-reviewer
description: Expert code reviewer. Use immediately after writing or modifying code.
tools: Read, Glob, Grep
model: sonnet
---

You are a senior code reviewer ensuring high standards of quality and security.

When invoked:
1. Run git diff to see recent changes
2. Focus on modified files
3. Begin review immediately
```

Only `name` and `description` are required. Everything after the closing `---` is the system prompt body. Sub-agents do **not** see the full Claude Code system prompt — only this body plus environment details.

## Frontmatter Fields

| Field | Required | What it does |
| :---- | :------- | :----------- |
| `name` | YES | Unique kebab-case id. Must be lowercase letters and hyphens only. |
| `description` | YES | Tells Claude when to delegate. State what and when. Use "use proactively" or "use immediately" to encourage automatic delegation. |
| `tools` | No | Comma-separated allowlist. Omit to inherit all tools from the parent. |
| `disallowedTools` | No | Denylist applied after `tools` resolution. |
| `model` | No | `sonnet`, `opus`, `haiku`, a full model id, or `inherit`. Default: `inherit`. |
| `permissionMode` | No | `default`, `acceptEdits`, `auto`, `bypassPermissions`, or `plan`. Ignored for plugin sub-agents. |
| `maxTurns` | No | Hard cap on agentic turns before the sub-agent stops. |
| `skills` | No | Skill names to preload at startup. Full skill content is injected — not just the description. |
| `mcpServers` | No | MCP servers scoped to this sub-agent. Ignored for plugin sub-agents. |
| `hooks` | No | Lifecycle hooks scoped to this sub-agent. Ignored for plugin sub-agents. |
| `memory` | No | `user`, `project`, or `local`. Enables a persistent memory directory. |
| `isolation` | No | `worktree` runs the sub-agent inside a temporary git worktree. Auto-cleaned if no changes. |
| `background` | No | `true` to always run as a background task. |
| `effort` | No | `low`, `medium`, `high`, `xhigh`, `max`. Overrides session effort. |
| `color` | No | UI display color: `red`, `blue`, `green`, `yellow`, `purple`, `orange`, `pink`, `cyan`. |

<Warning>
Plugin sub-agents ignore `hooks`, `mcpServers`, and `permissionMode`. Those fields are silently dropped. Copy the file into `.claude/agents/` or `~/.claude/agents/` if you need them.
</Warning>

## Writing the System Prompt Body

Write the body in second person ("You are a..."). Make the role specific and lay out a concrete workflow. A focused workflow with a checklist and a defined output format produces far better results than a vague role description.

```markdown
You are a senior code reviewer ensuring high standards of code quality and security.

When invoked:
1. Run git diff to see recent changes
2. Focus on modified files
3. Begin review immediately

Review checklist:
- Code is clear and readable
- No secrets or credentials are present
- Error paths are handled

Provide feedback organized by priority:
- Critical issues (must fix)
- Warnings (should fix)
- Suggestions (consider improving)
```

## Model Selection

Resolution order when Claude invokes a sub-agent:

1. `CLAUDE_CODE_SUBAGENT_MODEL` env var
2. Per-invocation `model` parameter from the Agent tool
3. Frontmatter `model:`
4. Main conversation's model

Use `haiku` for cheap, fast, low-stakes lookups. Use `sonnet` for review and analysis. Use `opus` for the hardest reasoning tasks. Default to `inherit` to match the main thread.

## Restart Requirement

Sub-agents are loaded at session start. If you create or edit a file directly on disk, restart the session to pick up changes. Files created through the `/agents` UI take effect immediately.
