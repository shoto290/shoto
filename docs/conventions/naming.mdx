---
title: "Naming Conventions"
description: "kebab-case files, name: matches path, Title Case headings — the portable naming rules for all shoto artifacts."
---

These rules apply to every file and artifact in the shoto repo. Violating them is a **blocking** enforced rule.

## Files and Directories

Use `kebab-case` for every file and directory: lowercase letters, digits, and hyphens. No spaces, no underscores, no camelCase.

| Type | Convention | Example |
| :--- | :--------- | :------ |
| Files | kebab-case | `git-safety.md`, `pr-summary.md` |
| Directories | kebab-case | `skill-architect/`, `review-fix/` |
| Skill / agent `name:` | kebab-case, matches path | `name: skill-architect` |
| Headings | Title Case | `## Typical Flow` |

## Skill and Agent `name:` Fields

The `name:` field in frontmatter must be `kebab-case` and must match the directory or filename that holds the artifact:

- A skill at `plugins/core/skills/foo/SKILL.md` must declare `name: foo`.
- A sub-agent at `plugins/core/agents/bar.md` must declare `name: bar`.

Renaming the directory or file without updating `name:` (or vice versa) breaks discovery and slash-command routing.

## Headings

Use Title Case for top-level and section headings. Keep heading depth shallow — prefer two or three levels over five.

| Do | Don't |
| :- | :---- |
| `## Typical Flow` | `## typical flow` |
| `## How to Extend` | `## HOW TO EXTEND` |

## Slash Commands

A skill's slash command is derived from its directory name unless overridden by the `name:` field in frontmatter. Keep names short and unambiguous — aim for one or two words. Plugin skills are namespaced as `<plugin>:<skill>`, so `/core:skill` and `/utils:explore` are valid commands.

<Note>
The full naming reference — including edge cases for plugin namespacing and scope collision rules — lives in the `core:base` skill's reference file at `plugins/core/skills/base/reference/naming.md`.
</Note>
