The shoto repo exists to build skills, sub-agents, and hooks for Claude Code. Each artifact type has a dedicated author that handles frontmatter, scope selection, and the validation gate. Bypassing the authors means losing those guardrails.

## Use the Dedicated Authors

| Artifact | Author |
| :------- | :----- |
| Skill (`plugins/<plugin>/skills/<name>/SKILL.md`) | `/core:skill` or the `skill-architect` sub-agent |
| Sub-agent (`plugins/<plugin>/agents/<name>.md`) | `/core:subagent` or the `subagent-architect` sub-agent |
| Hook (`.claude/hooks/*`) | `/core:hooks` skill |
| Coordinated multi-artifact change | `/core:evolve` (plans across skill / sub-agent / hook) |

The architects own frontmatter, scope selection, and the validation gate. Don't bypass them when scaffolding new artifacts.

## House Rules

Beyond using the dedicated authors, three house rules apply to every change:

- **Check before creating** — Search for existing skills, agents, and hooks before adding new ones. Reuse over duplication.
- **One concern per commit** — Each commit addresses a single logical change.
- **Keep the manifest in sync** — When adding or renaming a skill or agent, update the relevant `plugins/<plugin>/.claude-plugin/plugin.json` and `.claude-plugin/marketplace.json`.
- **Match `name:` to the path** — A skill at `plugins/<plugin>/skills/foo/SKILL.md` must declare `name: foo`. Same rule applies to agents.

<Warning>
Never push directly to `main`. Always work on a feature branch. See [Safety rules](/conventions/safety) for the full list of operations that require explicit confirmation.
</Warning>
