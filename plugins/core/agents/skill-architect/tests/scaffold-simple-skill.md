name: scaffold-simple-skill
description: Exercises skill-architect's ability to scaffold a minimal new skill from a one-line brief — dry run, no files written.
---

I want a new project-scope skill named `hello-world` that, when invoked as `/hello-world <name>`, returns the string `hello, <name>!`.

Describe the scaffold you would create:

- Exact file paths (relative to the repo root)
- The full content of `SKILL.md` (frontmatter + body)
- Your scope choice (user vs project) and one-sentence justification
- Whether any supporting files (`scripts/`, `references/`, `examples/`) are warranted, and why or why not

DO NOT write any files. This is a benchmark dry run — only the description matters.
